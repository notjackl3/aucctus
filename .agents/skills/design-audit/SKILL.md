# design-audit

## Use when
- Reviewing UI consistency, accessibility, and visual regressions.

## Workflow
1. Identify affected pages/components.
2. Check spacing, hierarchy, contrast, and interaction states.
3. Flag violations against existing design patterns.
4. Propose minimal fixes and implementation order.
