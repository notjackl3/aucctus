# Agent Component Compiler

Compiles React TSX components generated by the AI agent into browser-executable JavaScript with runtime scope injection.

## Overview

When the agent generates a TSX component (like `CompetitiveLandscape.tsx`), it can't be directly executed in the browser because:

1. Browsers don't understand TSX syntax
2. Import statements like `@components` can't be resolved at runtime
3. TypeScript types need to be stripped

This compiler solves all three problems by:

1. Using esbuild to transpile TSX → JavaScript
2. Replacing imports with references to scope variables injected at runtime
3. Wrapping the output in a factory function that receives the dependencies

## Verification

The compiler includes verification utilities to catch errors before sending to the frontend:

```typescript
import {
  compileComponent,
  verifyTSXSource,
  verifyCompiledCode,
  formatErrorForAgent,
} from '@aucctus/agent-compiler';

// 1. Verify source before compiling
const sourceCheck = verifyTSXSource(tsxCode);
if (!sourceCheck.valid) {
  console.error(formatErrorForAgent(sourceCheck));
  // Fix and retry
}

// 2. Compile
const result = await compileComponent(tsxCode);

// 3. Verify compiled output
const compiledCheck = verifyCompiledCode(result.code);
if (!compiledCheck.valid) {
  console.error(formatErrorForAgent(compiledCheck));
  // Fix and retry
}
```

See `docs/VERIFICATION_GUIDE.md` for complete error codes and fixes.

## Usage

### From TypeScript/Node.js

```typescript
import { compileComponent, compileComponentFromFile } from '@aucctus/agent-compiler';

// Compile from a string
const tsxCode = `
import React, { useState } from 'react';
import { Icon } from '@components';
import { cn } from '@libs/utils/react';

const MyComponent: React.FC = () => {
  const [count, setCount] = useState(0);
  return (
    <button onClick={() => setCount(c => c + 1)} className={cn('btn btn-primary')}>
      <Icon variant="plus" /> Count: {count}
    </button>
  );
};

export default MyComponent;
`;

const result = await compileComponent(tsxCode);
console.log(result.code); // Browser-ready JavaScript
console.log(result.metadata); // { originalSize, compiledSize, compileTime }

// Or compile from a file
const result = await compileComponentFromFile('./generated/MyComponent.tsx');
```

### Integration with FastAPI

```python
from fastapi import FastAPI, HTTPException
from fastapi.responses import Response
import subprocess
import json

app = FastAPI()

@app.get("/components/{component_id}")
async def get_compiled_component(component_id: str):
    """Serve a compiled component."""
    try:
        # Compile the component using the Node.js compiler
        result = subprocess.run(
            ["node", "-e", f"""
                import {{ compileComponentFromFile }} from './dist/index.js';
                const result = await compileComponentFromFile('./generated/{component_id}.tsx');
                console.log(JSON.stringify(result));
            """],
            capture_output=True,
            text=True,
            cwd="./agent"
        )
        
        if result.returncode != 0:
            raise HTTPException(500, f"Compilation failed: {result.stderr}")
        
        compiled = json.loads(result.stdout)
        
        return Response(
            content=compiled["code"],
            media_type="application/javascript",
            headers={
                "X-Compile-Time": str(compiled["metadata"]["compileTime"]),
                "X-Original-Size": str(compiled["metadata"]["originalSize"]),
                "X-Compiled-Size": str(compiled["metadata"]["compiledSize"]),
            }
        )
    except FileNotFoundError:
        raise HTTPException(404, f"Component {component_id} not found")
```

## How It Works

### 1. Import Rewriting

The compiler intercepts imports and replaces them with scope references:

```typescript
// Input
import React, { useState } from 'react';
import { Icon } from '@components';
import { cn } from '@libs/utils/react';
import { BarChart, Bar } from 'recharts';

// After compilation, these become references to injected scope:
// - React hooks come from the React scope
// - Icon comes from Components.Icon
// - cn comes from Utils.cn
// - BarChart, Bar come from Recharts
```

### 2. Scope Injection

The compiled output is wrapped in a factory function:

```javascript
(function(React, Components, Utils, Recharts) {
  // React hooks are extracted
  var useState = React.useState;
  var useEffect = React.useEffect;
  // ... etc
  
  // Component shortcuts
  var Icon = Components.Icon;
  var cn = Utils.cn;
  
  // The compiled component code
  // ...
  
  return ComponentName;
})
```

### 3. Frontend Execution

The frontend calls this factory with the actual modules:

```typescript
import React from 'react';
import * as Components from '@components';
import * as Utils from '@libs/utils/react';
import * as Recharts from 'recharts';

const factory = eval(compiledCode); // or new Function()
const Component = factory(React, Components, Utils, Recharts);

// Now <Component /> can be rendered!
```

## Scope Mappings

| Import Path | Scope Variable | Contains |
|-------------|----------------|----------|
| `react` | `React` | React library, hooks, Fragment, etc. |
| `@components` | `Components` | Icon, Button, Modal, toast, etc. |
| `@libs/utils/react` | `Utils` | cn utility function |
| `recharts` | `Recharts` | All Recharts components |

## API Reference

### `compileComponent(tsxCode, options?)`

Compiles TSX source code to browser-executable JavaScript.

**Parameters:**
- `tsxCode` (string): Raw TSX source code
- `options` (CompilerOptions): Optional configuration
  - `minify` (boolean): Minify output (default: false)
  - `sourcemap` (boolean): Generate source maps (default: false)

**Returns:** `Promise<CompilationResult>`
- `code` (string): Compiled JavaScript
- `sourceMap` (string?): Source map if requested
- `warnings` (string[]): Compilation warnings
- `metadata`: Compilation statistics

### `compileComponentFromFile(filePath, options?)`

Compiles a TSX file to browser-executable JavaScript.

### `validateCompiledCode(compiledCode)`

Performs a basic syntax check on compiled code.

### `analyzeComponent(tsxCode)`

Quick analysis of component structure without full compilation.

## Security Considerations

⚠️ **The compiled code will be executed in the user's browser.** Ensure:

1. The agent only generates trusted code
2. You validate/lint the TSX before compilation
3. Consider sandboxing with iframes if needed
4. Use CSP headers appropriately

## Troubleshooting

### "Module not found" errors

Ensure all imports use the supported aliases:
- `@components` ✅
- `@libs/utils/react` ✅
- `./relative/path` ❌ (not supported)

### "X is not defined" in browser

The component uses something not in the scope. Either:
1. Add it to the scope mappings in `compile.ts`
2. Or ensure it's exported from one of the scope modules
